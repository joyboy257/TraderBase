# TraderBase — Design Context

## Design Context

### Users
Retail traders — everyday people who want to discover, follow, and automatically copy verified successful traders. They value credibility (proof, not screenshots), clarity (data they can act on), and trust (their money is involved). Not Bloomberg power users — they want the social layer without the overwhelming complexity.

### Brand Personality
**Trustworthy & Precise** — Bloomberg meets Robinhood. Data-dense enough to be credible, clean enough to be approachable. Every element earns its place; no decoration for decoration's sake. The interface communicates: "real traders, real positions, real results."

### Aesthetic Direction
Modern fintech dark UI — bold accent colors on near-black surfaces with obsessive data presentation clarity. Reference: Afterpay's dark mode transaction UI. The green/purple neon accents are signal, not decoration — they carry meaning (buy/sell, follow/active).

### Design Principles
1. **Credibility over flair** — Every visual choice must reinforce trust. Verified badges, real brokerage links, honest numbers. No gradient hero backgrounds or decorative noise that screams "startup landing page."
2. **Data hierarchy is everything** — The most important information (price, action, P&L, trader name) must be instantly scannable. Dense but not cluttered.
3. **Neon is semantic, not decorative** — The green/purple palette carries meaning: BUY = green, SELL = red, VERIFIED = purple glow. Never use these colors decoratively.
4. **Motion with purpose** — GSAP animations exist to guide attention and confirm interactions, not to impress. Scroll reveals and micro-interactions should feel inevitable, not added.
5. **No AI slop patterns** — No gradient bento grids, decorative 3D renders, floating abstract blobs, or "proven results" badge walls. Every element must feel hand-crafted for this product.

### Tech & Design Stack
- **Theme:** Dark only (no light mode — this is a trading/fintech app)
- **Colors:** Deep dark bg (`#0a0a0f`), neon green (`#32ff48`), purple (`#6f2bff`), semantic red (`#ff4757`)
- **Fonts:** Instrument Serif (display/headlines), DM Sans (body), JetBrains Mono (prices/data)
- **Framework:** Next.js 14 App Router, Tailwind CSS v3 + CSS variables
- **Animations:** GSAP + ScrollTrigger for landing page choreography
- **Icons:** Lucide React (consistent stroke-based SVG, no emoji)

### Current Design System (established)
- `Button`: primary (green), secondary (bordered), ghost, danger variants; sm/md/lg sizes
- `Badge`: buy (green glow), sell (red glow), verified (purple glow), neutral
- `Card`: surface bg, subtle border, optional hover lift
- `Avatar`: rounded, size variants sm/md/lg/xl
- `Input`: dark tertiary bg, green focus ring
- Design tokens: all via CSS custom properties in `globals.css`

### Known Anti-Patterns (do not repeat)
- ❌ Emoji in UI — use Lucide icons instead
- ❌ Bare Tailwind class names for colors (`text-text-primary`, `bg-bg-surface`) — must use CSS variable syntax
- ❌ Decorative gradient borders on cards
- ❌ `duration-*` values not in Tailwind scale (e.g. `duration-250`)
- ❌ Undefined CSS variables in shadow/hover values
- ❌ Missing `color-scheme: dark` on `<html>`
- ❌ No `prefers-reduced-motion` support
